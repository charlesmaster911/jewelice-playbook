과장 업무 매뉴얼

쥬얼아이스 · 일하는 방식과 점검표
과장은 팀장의 방향을 받아 실행하고, 담당 업무를 완료·기록하며, 동료·고객·파트너에게 신뢰를 쌓습니다.

일하는 방식 5가지

  1. 매일 출근 직후 — 오늘 처리할 항목 우선순위 정리
  2. 매일 퇴근 직전 5분 — 한 일 · 미완료 · 내일 첫 1개 정리
  3. 약속한 일은 명확한 마감일과 함께 — 혼자 외운 약속은 30%만 지켜집니다. 적은 약속·서면 약속은 70~90%
  4. 매번 만나는 사람에게 — "이 사람을 만나면 무언가 도움이 됐다"는 인상 남기기
  5. 막힐 때 — 30분 이상 혼자 헤매지 말고 팀장에게 도움 요청

꼭 필요한 도구 2개

1) Type C 목표 — 정말 원하는 결과 적기

"가능할 것 같은 것"이 아니라 "진짜 원하는 결과"를 적습니다.

"어떻게"는 묻지 마세요. WHAT만 명확히. 어떻게는 시작하면 보입니다.

매일 출근 후 첫 1줄: "오늘 정말 원하는 결과 = ___"

2) 안다·한다 통합 (Praxis)

"안다"는 것과 "한다"는 것은 전혀 다릅니다. 모두가 알고 있지만 대부분이 안 합니다.

아는 것 → 행동으로 바꾸는 두 가지 방법:

  • 매일 같은 시간 반복 — 새 습관은 같은 시간에 시작해야 박힙니다
  • 강한 감정으로 결정 — "꼭 한다"는 감정 없이는 패턴이 안 바뀝니다

한계 안 긋는 자세

"이건 제가..." / "보통 이렇게 하니까..." / "안 될 것 같은데" 멘트가 자주 나오면 자기검열 신호입니다.

대표·팀장이 직접 "하지 마"라고 하지 않은 한, 일단 시도해 봅니다.

실패해도 배움이 남습니다. 시도조차 안 한 것은 0이 남습니다.

매주 금요일 — 점검 5문

  • 이번 주 핵심을 한 문장으로 정리할 수 있나?
  • 약속한 일을 마감 안에 끝냈나? 못 했다면 사전에 알렸나?
  • 이번 주 누구에게 도움이 됐나? 이름 1명 떠올려 보기
  • 못 한다고 미리 그어버린 게 없나?
  • 다음 주 첫 한 걸음이 명확한가?

막힐 때 — 신호 보내기

  • 일반 진행 → CLAUDE.md 통해 자동 로그됨
  • 블로커 → 30분 이상 막히면 "블로커: ___ 왜냐하면 ___" 형식으로 팀장 보고
  • 실수 / 사고 → 숨기지 말고 즉시 보고. 빨리 알면 빨리 수습됨
