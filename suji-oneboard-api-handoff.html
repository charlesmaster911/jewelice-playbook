OneBoard API 연동 작업

수지야, 이 작업은 외부 문서 오가지 않고 OneBoard 안에서 전부 끝낼 수 있게 만들어 뒀어. 각 플랫폼에서 키를 받으면 OneBoard 설정 화면에 바로 붙여넣고, 카페24는 버튼 한 번으로 OAuth 자동 처리 돼. 찰스 거치지 않아도 혼자서 완결 가능.

📖 핵심 작업 흐름 — 6개 플랫폼 모두 공통
1
OneBoard 접속 → ⚙️ 설정 탭 → 비밀번호 JEWELICE 입력
한 번 입력하면 브라우저에 저장됨. 다시 안 물어봄.
2
각 플랫폼 카드에서 "🔑 키 입력" 버튼 클릭
팝업이 열리면 플랫폼에서 발급받은 키를 필드에 붙여넣기만 하면 됨.
3
저장하면 서버에서 자동 암호화되어 저장
키는 브라우저에 저장되지 않고 서버 암호화 DB에만 저장. 안전함.
4
카페24만 특별 — 키 저장 후 "🔗 OAuth 인증 시작" 버튼 추가로 클릭
버튼 누르면 카페24 로그인 페이지로 자동 이동 → "허용" 클릭하면 끝. access_token/refresh_token은 자동 저장됨.
5
플랫폼 카드 상태가 🟢 연결됨으로 바뀌면 완료
나머지 플랫폼도 같은 방식으로 반복.

🛒 P2 · 카페24 (자사몰)

OAuth 자동 처리
P2

3개 키 저장 → OAuth 버튼 1회 클릭 = 완료

20분 · ⭐⭐
💡 이 작업의 특징: 과거에는 수동 OAuth 플로우 필요했지만, 이제는 OneBoard 설정에서 "🔗 OAuth 인증 시작" 버튼 한 번 누르면 카페24 로그인 페이지로 자동 이동 → "허용" 클릭 → 자동 복귀. access/refresh 토큰은 서버가 알아서 받아 저장합니다.
OneBoard 조작 순서:
1) 설정 → 카페24 카드 → 🔑 키 입력 → mall_id + client_id + client_secret 3개 입력 → 저장
2) 같은 카드에서 🔗 OAuth 인증 시작 버튼 클릭 → 카페24 페이지로 이동
3) 카페24 로그인 후 "허용" 클릭 → 자동으로 OneBoard로 돌아옴 → ✅ 연결됨

1단계: 카페24 개발자센터에서 앱 등록

1
🔗 카페24 개발자센터 접속 → 쥬얼아이스 운영자 로그인
2
"앱 만들기" → Public App (공개 앱) 선택
3
앱 기본 정보:
• 앱 이름: OneBoard 매출 연동
• Redirect URI: https://oneboard-server.onrender.com/api/admin/oauth/cafe24/callback
• 앱 종류: 일반 앱
4
Scopes ✓ mall.read_order · mall.read_product · mall.read_analytics · mall.read_store · mall.read_application
5
저장 → 앱 상세 페이지에서:
• Client ID 복사
• Client Secret 복사
• mall_id: 자사몰 URL 앞부분 (예: jewelice.cafe24.com → jewelice)

2단계: OneBoard에서 입력 + OAuth

6
OneBoard 설정 → 카페24 카드 → 🔑 키 입력 → 3개 값 입력 → 저장
7
같은 카드에서 🔗 OAuth 인증 시작 클릭
8
카페24 페이지에서 "허용" 클릭 → 자동 복귀 → ✅ 완료
⚠️ 주의: Redirect URI는 정확히 https://oneboard-server.onrender.com/api/admin/oauth/cafe24/callback으로 카페24 앱 설정에 등록해야 합니다. 한 글자라도 다르면 OAuth 인증이 거부됩니다.
